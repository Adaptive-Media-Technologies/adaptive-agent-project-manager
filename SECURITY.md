# Security Policy

Thanks for helping keep Agntive.ai and its users safe.

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Report security issues via either of the following private channels:

- **Email**: `hello+security@agntive.ai`
- **GitHub Security Advisories**: Use GitHub's private vulnerability reporting (Security tab → "Report a vulnerability") if enabled for this repository.

If you're unsure whether something is a security issue, **report it anyway**.

## What to Include

To help us triage quickly, include:

- A clear description of the issue and potential impact
- Steps to reproduce (proof-of-concept code is welcome)
- Affected components, routes, files, or endpoints (if known)
- Your environment details (browser/OS/version) when relevant
- Any relevant logs, screenshots, or HAR files (remove secrets)

## Response Targets

We aim to:

- **Acknowledge** your report within **2 business days**
- Provide a **status update** within **5 business days**

Complex issues may take longer to fully resolve, but we’ll keep you updated.

## Supported Versions

Security fixes are provided for the **latest release**. We generally do not backport fixes to older versions.

## Coordinated Disclosure / Safe Harbor

We support coordinated vulnerability disclosure and will not take legal action against researchers who:

- Make a good-faith effort to avoid privacy violations, data destruction, and service disruption
- Only access data necessary to demonstrate the issue
- Do not use social engineering, phishing, or physical attacks
- Give us a reasonable time to remediate before any public disclosure

If you accidentally access sensitive data, stop immediately and include details in your report so we can investigate and remediate.

## Severity

When possible, we prioritize issues using common severity frameworks (e.g., CVSS) and the real-world impact on users.
